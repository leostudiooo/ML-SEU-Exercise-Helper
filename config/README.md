# 关于`boundaries`的一些说明

除了`ml-base_track.json`外，这里的数据均来自抓包和通过官方的API请求得到，请勿改动。

`ml-base_track.json`文件中内容为基础的轨迹定位点，使用网上工具打点获得，精度较低(约为`10`厘米)，实际上传轨迹会加以一点的随机模糊。

> **一些合理性说明**：  
> GPS经纬度小数位数`7`位左右对应的实际距离误差约为`10`厘米。  
> 由于官方接口需要上传数据是保留`14`为小数的(`1`纳米的精度)，故我采用`7`为小数作为底数，加上从第`8`为开始的模糊，这样只有至多`20`厘米的偏差，完全可以过通过轨迹合法性检测。

注：可能随着系统变动，数据会有所更改。如有，请及时提`issues`，谢谢。
